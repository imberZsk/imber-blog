`1. 熟练掌握 TailwindCSS、TypeScript 类型体操、Web 安全（XSS、CSRF）、移动端自适应布局、响应式布局`

1.1 TailwindCSS ✅

用 tailwind 做了很多项目，从一开始的滥用[]，到现在的优雅的无 CSS 复用，它有很多好处，比如原子化，减小 CSS 体积，提高 CSS 复用性，提高开发效率，规范化 UI 开发

1.2 TypeScript ✅

TS静态类型检查，在编辑器中提前发现错误，减少生产环境问题，但是只做业务的话还是不要用太深入，权衡开发成本和维护成本
为什么 TS 类型编程可以叫类型体操，因为它是图灵完备的​，递归、条件判断、循环（映射类型）、变量（泛型）​，甚至能实现 ​加减乘除运算​

1.3. XSS、CSRF ✅

做魅族社区，魅族商城发帖评论会遇到这些问题，也手写过这两种攻击来模拟效果，所以对这两种攻击有一定的了解

XSS 主要出现在输入类型的需求，需要查找 dangerousSetInnerHTML/v-html/innerHTML 这种可能会出现的地方，防范的方案有 httpOnlyCookie/secure/CSP，字符转义

CSRF 首先后端应该遵循 Rest 的规范，get 不能改变数据状态，防范的方案有 samesite / csrfToken / 验证 Referer

XSS 的转换：比如遇到过文本里有图片的，需要正则匹配单独渲染

1.4. 自适应 & 响应式 ✅

做过很多魅族的营销活动，对于自适应和响应式有比较深入的研究

自适应使用 Rem + VW + Tailwind 比较方便，原理是比例关系，响应式则要看情况，简单的响应式用 flex/grid/百分比 + 媒体查询就可以实现，复杂的不方便过渡的需要自适应 + 媒体查询

需要特别注意的是图片的响应式，对于相同的图片使用 sizes + srcset 来实现，对于不同的图片使用 picture + media + srcset 来实现
